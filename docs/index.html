Connect Claude AI to Hospitable

Free setup guide. 6 steps. About 15 minutes.

What you need before starting

  • A Hospitable account with API access
  • A Claude account at claude.ai (free or paid)
  • A Railway account at railway.com (free signup, ~$5/mo)
1
Get Your Hospitable Token 2 min

This token lets the server access your Hospitable data. Think of it like a password just for the API.

  1. Go to my.hospitable.com
  2. Click Apps in the left sidebar
  3. Click API access
  4. Click Access tokens
  5. Click + Add new
  6. Select Read and Write permissions
  7. Enter your password and confirm
  8. Copy the token and save it somewhere (you'll need it in Step 4)
Screenshot: Hospitable token creation page
Heads up: This token expires after about 1 year. If things stop working later, just come back and generate a new one.
2
Create a Railway Account 2 min

Railway is where your server will run. It stays on 24/7 so you can use Hospitable from Claude anytime, on any device.

  1. Go to railway.com
  2. Click Sign Up (GitHub login is fastest)
  3. You may need to add a credit card for the Hobby plan (~$5/mo)
Screenshot: Railway signup page
3
Deploy the Server 3 min

This copies the server code to Railway and starts it up.

  1. In Railway, click New Project
  2. Click Deploy from GitHub Repo
  3. Paste this URL:
https://github.com/FBRMike/hospitable-mcp
  1. Wait about 30 seconds for the build to finish
Screenshot: Railway "Deploy from GitHub Repo" dialog
4
Configure the Server 5 min

Three quick settings in the Railway dashboard.

A) Add your Hospitable token:

  1. Click on your service in the Railway dashboard
  2. Click the Variables tab
  3. Click + New Variable
  4. Name: HOSPITABLE_PAT
  5. Value: paste your token from Step 1
  6. Click Add
Screenshot: Railway Variables tab with HOSPITABLE_PAT

B) Set the start command:

  1. Click the Settings tab
  2. Scroll down to the Deploy section
  3. Click + Start Command
  4. Type exactly:
python server.py --http
Screenshot: Railway start command field

C) Enable public access:

  1. Still in Settings, scroll to Networking
  2. Under Public Networking, click Generate Domain
  3. You'll get a URL like: your-service-production-xxxx.up.railway.app
  4. Copy this URL. You'll need it next.
Screenshot: Railway generated domain URL

Click Deploy at the bottom to apply all changes. Wait for the deployment to show ACTIVE in the Deployments tab.

5
Connect to Claude 2 min

This is the last step. You're telling Claude where to find your Hospitable server.

  1. Go to claude.ai and log in
  2. Click Settings (bottom left corner)
  3. Click Connectors
  4. Scroll down and click Add custom connector
  5. Name: Hospitable
  6. URL: paste your Railway URL from Step 4C, then add /sse at the end
https://your-service-production-xxxx.up.railway.app/sse
  1. Click Add
  2. Find Hospitable in the list and click Connect
If the button changes from "Connect" to "Configure", you're connected!
Screenshot: Claude.ai custom connector dialog + connected state
6
Test It 1 min

Open a new chat in Claude and try:

Use the Hospitable tools to get my account info

Claude should respond with your Hospitable account details. If it works, try these:

Check-ins: "Show me upcoming check-ins"
Reviews: "What reviews came in this month?"
Messages: "Pull up guest messages for [property]"
Calendar: "What's my July calendar look like?"
Pricing: "Update pricing to $300/night for July 4th"
Properties: "List all my properties"

Troubleshooting

"Connect" button doesn't change to "Configure": Make sure your Railway deployment shows "ACTIVE" (not crashed). Make sure you added /sse to the end of the URL. Make sure the start command is exactly: python server.py --http

Railway deployment keeps crashing: Check that HOSPITABLE_PAT is set correctly in Variables with no extra spaces.

Claude can't find Hospitable tools: Go to Settings, Connectors, and verify Hospitable shows "Configure." Start a new chat (tools don't appear mid-conversation).

Token expired / 401 errors: Generate a new PAT at my.hospitable.com and update the HOSPITABLE_PAT variable in Railway. It'll auto-redeploy.
